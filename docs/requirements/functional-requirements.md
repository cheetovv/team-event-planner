# Functional Requirements

## RF-01
El usuario debe poder iniciar sesión.

## RF-02
Un coordinador debe poder crear eventos.

## RF-03
Un usuario debe poder aceptar una invitación.

## RF-04
Un usuario debe poder rechazar una invitación.

## RF-05
El sistema debe mostrar el calendario mensual.

## RF-06
El sistema debe detectar conflictos de horario.

## RF-07
Un usuario puede cambiar de opinion sobre un evento siempre y cuando no haya iniciado
PENDIENTE -> ACEPTADA
PENDIENTE -> RECHAZADA

ACEPTADA -> RECHAZADA
RECHAZADA -> ACEPTADA

## RF-08
No se pueden crear nuevas invitaciones si el evento ya ha comenzado o ha terminado

## RF-09
Cuando un equipo cambia de coordinador, los eventos se mantienen con su creador original, el nuevo coordinador administra esos eventos,
el nuevo coordinador puede mantener o eliminar esos eventos.

## RF-10
Cuando un usuario abandona la organizacion se mantiene inactivo, sin eliminar.
